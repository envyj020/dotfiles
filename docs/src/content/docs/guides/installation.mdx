The installation is an interactive process. Whether using the standalone installer or the manual method,
you will need to provide your own preferences.

<Aside icon="magnifier">
  [GNU Stow](https://www.gnu.org/software/stow) is used to manage symlinks and
  it might fail to do so if it finds an existing file such as a default `~/.bashrc`
  and `~/.profile` where it's trying to create a symlink. Ensure any conflicting files
  are backed up to a non-standard location before retrying.
</Aside>

## Prerequisites

<Aside type="caution" title="Watch out!" icon="github">
  The installation relies on the GitHub API and because unauthenticated requests
  are rate limited at 60 requests/hour, the setup will likely fail without a
  token. To ensure a smooth installation, please generate a [Github
  PAT](https://github.com/settings/tokens/new?description=DOTFILES_TOKEN), with
  no scopes required and export it as `GITHUB_TOKEN` before
  running any of the methods outlined below:
  ```sh
  export GITHUB_TOKEN=$(<"$HOME/.GITHUB_TOKEN")

  ```
</Aside>

## Installation


<Tabs>
  <TabItem label="Standalone Installer" icon="rocket">
    <Code
      code="sh <(curl -sSfL https://envyj020.github.io/dotfiles/run)"
      lang="sh"
    />
    <Aside type="tip">
      The standalone installer may be inspected before use or alternatively
      check out the source code directly on [GitHub](https://github.com/envyj020/dotfiles/blob/main/docs/public/run).
      <Code code="curl -sSfL https://envyj020.github.io/dotfiles/run | less" lang="sh"/>
    </Aside>
  </TabItem>
  <TabItem label="Manual Installation" icon="seti:powershell">
    <Steps>
      1. Clone the repository:
          ```sh
          git clone --depth 1 --recursive https://github.com/envyj020/dotfiles.git
          cd dotfiles
          ```
      2. Install the OS baseline packages:
          ```sh
          make base
          make mise
          ```
      3. Symlink dotfiles settings:
          ```sh
          make stow
          ```
    </Steps>
  </TabItem>
</Tabs>

## Post-Installation

Congratulations! Youâ€™ve successfully installed this desktop environment.
To ensure your workspace looks and behaves correctly after a reboot, please complete the following steps:

### 1. Optional: Set your wallpaper

By default, i3 and/or Sway look for their wallpaper at `~/Pictures/wallpaper.jpg`. While a default one is provided
with the standalone installer method, you can set your own by replacing it.

### 2. Optional: API Tokens

API access for certain services requires storing authentication token files under
your home directory. The provided `.bashrc` exports these variables by reading them
if they exists and keeps your sensitive keys out of your shell history and configuration files.

The table below outlines the expected filenames:

<div class="overview-table">

| Env var             | Filename             | Purpose                                      |
| ------------------- | -------------------- | -------------------------------------------- |
| `GITHUB_TOKEN`      | `~/.GITHUB_TOKEN`    | Github CLI                                   |
| `TINYURL_TOKEN`     | `~/.TINYURL_TOKEN`   | Bash URL `shortener` alias                   |
| `GEMINI_API_KEY`    | `~/.GEMINI_TOKEN`    | Gemini CLI and Neovim Avante Gemini provider |
| `ANTHROPIC_API_KEY` | `~/.ANTHROPIC_TOKEN` | Neovim Avante Claude provider                |

</div>

### 3. Optional: Change i3/Sway terminal

While Alacritty is set as the default term, you can switch to your preferred terminal by modifying
the `$TERMINAL` environment variable. This will automatically update the terminal launched by
the window manager `mod+Enter` shortcut.

```sh title="~/.bashrc.d/00-EXPORTS" del={2} ins={3}
...
export TERMINAL=alacritty
export TERMINAL=kitty
...
```

### 4. Configure your display layout

<Tabs>
  <TabItem label="i3" icon="external">
    Open your terminal and run the following command once you have your
    monitors arranged the way you like them:
    <Code code="autorandr --save default" lang="sh" />
    <Aside type="tip" title="autorandr">
      `autorandr` was installed as part of the OS baseline. The command above will
      save your monitor setup so i3 will know which settings to apply upon startup.
    </Aside>
  </TabItem>
  <TabItem label="Sway" icon="pen">
    Yet to spot a Wayland-native alternative that effectively replicates
    the display management workflow provided by arandr and autorandr on x11. For now
    outputs must be identified and applied right into Sway's configuration:
    <Steps>
      1. Identify your output/s:
          ```sh
          swaymsg -t get_outputs | jq -r '.[] | {name} + .rect | "\(.x),\(.y) \(.width)x\(.height) \(.name)"'
          ```
      2. Open Sway's config and adjust your output/s to fit your needs:
          ```sh title="~/.config/sway/config" 'eDP-1' '1920x1080@120Hz' '0,0'
            ...
            output "eDP-1" mode 1920x1080@120Hz position 0,0
            ...
          ```
      3. Reload Sway config with `$mod+Shift+c`
    </Steps>
  </TabItem>
</Tabs>
