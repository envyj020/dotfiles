The installation is an interactive process. Whether using the standalone installer or the manual method,
you will need to provide your own preferences.

<Aside icon="magnifier">
  [GNU Stow](https://www.gnu.org/software/stow) is used to manage symlinks and
  it might fail to do so if it finds an existing file (like a default .bashrc)
  where it's trying to create a symlink. You may skip this step; otherwise,
  ensure any conflicting files are backed up to a non-standard location before
  re-running any of the outlined methods below.
</Aside>

  <TabItem label="Manual Installation" icon="seti:powershell">
    <Steps>
      1. Clone the repository:
          ```sh
          git clone --depth 1 --recursive https://github.com/envyj020/dotfiles.git
          cd dotfiles
          ```
      2. Install the OS baseline packages:
          ```sh
          make base
          make mise
          ```
      3. Symlink dotfiles settings:
          ```sh
          make stow
          ```
    </Steps>
  </TabItem>
</Tabs>

## Post-Installation

Congratulations! Youâ€™ve successfully installed this desktop environment.
To ensure your workspace looks and behaves correctly after a reboot, please complete the following steps:

### 1. Optional: Set your wallpaper

By default, i3 looks for its wallpaper at `~/Pictures/wallpaper.jpg`. While a default one is provided
with the standalone installer method, you can set your own by overriding it.

### 2. Optional: API Tokens

API access for certain services requires storing authentication token files under
your home directory.  The provided `.bashrc` exports these variables by reading them
if they exists and keeps your sensitive keys out of your shell history and configuration files.

The table below outlines the expected filenames conventions:

<div class="overview-table">

| Env var             | Filename             | Purpose                                      |
| ------------------- | -------------------- | -------------------------------------------- |
| `GITHUB_TOKEN`      | `~/.GITHUB_TOKEN`    | Github CLI                                   |
| `TINYURL_TOKEN`     | `~/.TINYURL_TOKEN`   | Bash URL `shortener` alias                   |
| `GEMINI_API_KEY`    | `~/.GEMINI_TOKEN`    | Gemini CLI and Neovim Avante Gemini provider |
| `ANTHROPIC_API_KEY` | `~/.ANTHROPIC_TOKEN` | Neovim Avante Claude provider                |

</div>

### 3. Optional: Change i3 terminal

While Alacritty is the standard default, you can switch to another terminal such as kitty by modifying
the `$TERMINAL` environment variable. This will automatically update the terminal launched by the i3 `mod+Enter` shortcut.

```sh title="~/.bashrc.d/00-EXPORTS" del={2} ins={3}
...
export TERMINAL=alacritty
export TERMINAL=kitty
...
```

### 4. Configure your display layout

Open your terminal and run the following command once you have your monitors arranged the way you like them:

<Code code="autorandr --save default" lang="sh" />
<Aside title="autorandr">
  `autorandr` was installed as part of the OS baseline. The command above will
  save your monitor setup so i3 will know which settings to apply upon startup.
</Aside>
